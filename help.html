Is the AI slow? — Troubleshooting Guide

This app runs AI models directly in your browser. Performance depends on which engine is active and how your system is configured. Below are the most common causes of slow inference and how to fix them.

WebGPU / web-llm

Using the wrong GPU

On systems with multiple GPUs (e.g. a laptop with both Intel integrated and NVIDIA discrete graphics), Windows may route the browser to the slower integrated GPU by default. This is the most common reason for slow WebGPU inference.

How to check

Look at the GPU dropdown in the chat header. The label after "→ using:" shows which GPU the browser is actually using. If it says something like intel instead of your discrete GPU, the browser is using the wrong one.

Fix: Windows Graphics Settings

  1. Open Settings → System → Display → Graphics (or search for "Graphics settings" in the Start menu).
  2. Click "Browse" and find your browser executable:
    • Chrome: C:\Program Files\Google\Chrome\Application\chrome.exe
    • Edge: C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe
  3. Click the browser entry, then "Options".
  4. Select "High performance" and click Save.
  5. Restart the browser for the change to take effect.
Tip: After changing this, reload the app and check the GPU label again. It should now show your discrete GPU (e.g. nvidia).

Fix: NVIDIA Control Panel

If you have an NVIDIA GPU, you can also force it per-application:

  1. Right-click the desktop → NVIDIA Control Panel.
  2. Go to Manage 3D settings → Program Settings.
  3. Add your browser and set "Preferred graphics processor" to "High-performance NVIDIA processor".

Model is too large

Larger models need more VRAM and take longer to run. If inference is slow, try a smaller model from the dropdown. Models marked with ★ are already downloaded and will load instantly. Look for models with lower VRAM requirements (shown in MB next to each model name).

Recommended small models: SmolLM2-360M or Qwen2.5-0.5B variants are fast even on integrated GPUs. Gemma-2-2B is a good balance of quality and speed on discrete GPUs.

First-time model download

The first time you use a model, it needs to be downloaded (typically 200 MB – 6 GB depending on the model). This is a one-time cost — subsequent loads use the browser cache. You can see the download progress in the chat status area.

Chrome Built-in AI (Prompt API)

Setting up Chrome's built-in AI

Chrome 138+ includes a built-in AI model (Gemini Nano) that runs locally on your device. When available, this is the fastest option since it's deeply integrated into the browser. If it's not set up, the app falls back to WebGPU (web-llm) which requires downloading a separate model.

How to enable it

  1. Make sure you're running Chrome 138 or newer. Check your version at chrome://version.
  2. Navigate to chrome://flags/#optimization-guide-on-device-model and set it to "Enabled BypassPerfRequirement".
  3. Navigate to chrome://flags/#prompt-api-for-gemini-nano-multimodal-input and set it to "Enabled".
  4. Relaunch Chrome when prompted.
  5. Wait for the model to download. You can check the status at chrome://on-device-internals.
Note: The model download happens in the background and can take several minutes. The Prompt API won't work until the download is complete. Check chrome://on-device-internals to see if the model status shows "Ready".

Why is Chrome AI not being used?

If the app shows "(via WebGPU)" in timestamps or you see the model selector dropdown, it means Chrome's Prompt API was not detected. Common reasons:

  • Chrome version is older than 138.
  • The flags above are not enabled.
  • The on-device model hasn't finished downloading yet.
  • You're using a different browser (Edge, Firefox) — the Prompt API is Chrome-only.

General performance tips

  • Close other GPU-heavy tabs — video streaming, games, or other WebGPU apps compete for GPU memory.
  • Keep the browser updated — WebGPU performance improves with each browser release.
  • Check your GPU drivers — outdated drivers can cause slow or broken WebGPU.
  • Use a Chromium-based browser — Chrome and Edge have the most mature WebGPU implementations.
